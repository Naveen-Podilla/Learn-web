Introduction to Angular

1. Angular Components

Angular components are the fundamental building blocks of an Angular application. Each component is a TypeScript class decorated with the @Component decorator. Components consist of a template, a class, and styles.

Component Example

@Component({
    selector: 'app-hello-world',
    template: '<h1>Hello, {{name}}!</h1>',
    styles: ['h1 { color: #0056b3; }']
})
export class HelloWorldComponent {
    name: string = 'World';
}

Output:

Rendered HTML:

<h1>Hello, World!</h1>

2. Angular Templates

Templates define the view layout for Angular components. They use Angular's template syntax to bind data to HTML elements.

Template Binding Example

@Component({
    selector: 'app-user-profile',
    template: `
        <div>
            <h2>User Profile</h2>
            <p>Name: {{user.name}}</p>
            <p>Age: {{user.age}}</p>
        </div>
    `,
    styles: ['div { font-family: Arial, sans-serif; }']
})
export class UserProfileComponent {
    user = { name: 'Alice', age: 30 };
}

Output:

Rendered HTML:

<div>
    <h2>User Profile</h2>
    <p>Name: Alice</p>
    <p>Age: 30</p>
</div>

3. Angular Modules

Modules in Angular organize an application into cohesive blocks of functionality. Each Angular application has at least one module, the root module, which is usually named AppModule.

Module Example

@NgModule({
    declarations: [HelloWorldComponent, UserProfileComponent],
    imports: [BrowserModule],
    providers: [],
    bootstrap: [AppComponent]
})
export class AppModule { }

Explanation:

This module imports the BrowserModule, declares two components, and specifies the AppComponent as the bootstrap component.

4. Data Binding

Data binding in Angular allows for the synchronization between the model and the view. There are several types of data binding:

  • Interpolation: {{ value }}
  • Property Binding: [property]="value"
  • Event Binding: (event)="handler"
  • Two-Way Binding: [(ngModel)]="value"

Data Binding Example

@Component({
    selector: 'app-data-binding',
    template: `
        <input [(ngModel)]="name" />
        <p>Hello, {{name}}!</p>
    `,
    styles: ['input { margin-right: 10px; }']
})
export class DataBindingComponent {
    name: string = 'World';
}

Output:

Rendered HTML:

<input value="World">
<p>Hello, World!</p>

Typing in the input field updates the text immediately due to two-way binding.

5. Directives

Directives are classes that add additional behavior to elements in your Angular applications. There are three types of directives:

  • Components: Directives with a template.
  • Structural Directives: Change the DOM layout by adding or removing elements (e.g., *ngIf, *ngFor).
  • Attribute Directives: Change the appearance or behavior of an element (e.g., [ngStyle], [ngClass]).

Structural Directive Example

<ul>
    <li *ngFor="let item of items">{{item}}</li>
</ul>

<!-- In component class -->
export class ListComponent {
    items: string[] = ['Item 1', 'Item 2', 'Item 3'];
}

Output:

Rendered HTML:

<ul>
    <li>Item 1</li>
    <li>Item 2</li>
    <li>Item 3</li>
</ul>

6. Services

Services are used to share data and logic across multiple components. They are typically used for data access, business logic, and state management.

Service Example

@Injectable({
    providedIn: 'root'
})
export class UserService {
    getUser() {
        return { name: 'Alice', age: 30 };
    }
}

@Component({
    selector: 'app-user',
    template: '<p>User: {{user.name}}, Age: {{user.age}}</p>',
})
export class UserComponent {
    user: any;

    constructor(private userService: UserService) {
        this.user = this.userService.getUser();
    }
}

Output:

Rendered HTML:

<p>User: Alice, Age: 30</p>
